### __3. Характеристика объектов автоматизации__

__3.1 Объект автоматизации__

        Объектом автоматизации является процесс распросранения книжной продукции в 
        электронном виде, процесс покупки книжной продукции. Распространителями продукции являются либо 
        издательство (компания, проверенная системой, имеющая юридические права на выпуск продукции), либо автор 
        (лицо, проверенное системой, т.е. имеющее уже выпущенную или подтвержденную, но еще не выпущенную продукцию). 
        Издательство/автор проводит компанию по распространения своей продукции, 
        доводит информацию об издании до максимального числа потенциальных покупателей. Покупатель
        (лицо, заинтресованное в покупке продукции) приобретает необходимую ему продукцию

# Предметная область и требования к системе

### __Предметная область__

Предметная область - распространение и покупка книг.

Неавторизованный пользователь заходит на сайт. Может просмотреть список всех книг, жанров, авторов, издательств.
При поиске книг пользователь может производить поиск по различным параметрам (комбинациям параметров): наименование,
автор, издательство, год издания, жанр, год написания, статус издания, возрастное ограничение и тд. Поиск автора, 
издания, жанров производится по наименованию. Пользователь может перейти на любую книгу, увидеть о ней подробную 
информацию, отзывы читателей с оценками, может перейти на автора книги, издательство, жанр. При просмотре авторов 
пользователь может перейти на любого из них, увидеть подробную информацию о нем, отзывы читателей с оценками, список самых 
популярных книг. При просмотре издательств пользователь может перейти на одно из, увидеть подробную информацию о нем,
список самых популярных книг. При просмотре жанров пользователь может перейти на один из них, увидеть список книг данного
жанра. Если пользователь хочет добавить книгу себе в корзину, он должен авторизоваться/зарегистрироваться в системе.

Авторизованный пользователь может выйти из аккаунта, изменить пароль. Может добавить/удалить книгу в свою корзину. Для оформления 
покупки пользователь обязан подтвердить свой e-mail. После подтверждения своего e-mail, пользователь может оплатить заказ. При 
успешной оплате книги переносятся из корзины в личную библиотеку. Из личной библиотеки пользователь может пользоваться 
купленными книгами, следить за книгой, купленной предзаказом. Также авторизованный пользователь может оставить отзыв
на купленную книгу, автора купленной книги, издательство купленной книги. Если книга, автор ил издательство нарушают 
какие-либо правила (моральные, законодательство и тд.), пользователь в праве отправить жалобу на нарушение. Также 
пользователи могут отмечать добавлять авторов и жанры в избранное.

Если пользователь является автором, он может подать заявку на установку статуса автора. Предоставляя необходимое 
подтверждение личности и подтверждение наличия произведений, на которые у него есть права, пользователь получает статус 
автора. 

Если пользователь является лицом представляющим какое-либо издательство, он может подать заявление на регистрацию 
издательства в системе. Регистрируясь пользователь обязан предоставить необходимые документы доказывающие, его
связь с издательством, а также документы подтверждающие существование данного издательства, должен указать,
пользователя в системе, который будет уполномочен управлять произведениями, выпущенными данным издательством.
После проверки доказательств, издательство регистрируется в системе, пользователь, указанный в заявке получает 
статус уполномоченного лица издательства.

Пользователи, являющиеся уполномоченным лицом какого-либо издательства или автором, могут подавать заявки на публикацию
произведений, предоставляя все необходимые подтверждения права на распространение данного произведения. Также в заявке
указываются такие параметры как: цена без наценки, название, автор, указание, если автор есть в системе, жанр, описание,
доступ к электронной версии книги и тд. После подтверждения книга публикуется в системе. Автор/лицо уполномоченное
издательством могут просматривать статистику по покупке своих произведений, могут подавать заявки на блокировку своих
произведений, изменять (МБ ТУТ ТОЖЕ НУЖНА ЗАЯВОЧНАЯ СИТЕМА) некоторые параметры книги в системе.

Администратор имеет возможность рассмотрения всевозможных заявок пользователей. После рассмотрения заявок, может 
отклонить или принять их (в том числе и жалобы), имеет доступ к расширенной информации аккаунтов пользователей. 
Имеет доступ к расширенной информации об авторах, издательствах, покупках, отзывах
